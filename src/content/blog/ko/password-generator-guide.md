## 왜 강력한 비밀번호가 필요한가?

### 해킹 시도 통계
- 초당 수백만 개의 비밀번호 조합 시도 가능
- "123456"은 1초 미만에 해킹
- 8자 소문자만: 약 2시간
- 12자 혼합: 수백 년 이상

### 유출 피해
- 개인정보 도용
- 금융 피해
- 명예 훼손
- 다른 계정까지 연쇄 피해

## 강력한 비밀번호 조건

### 필수 요소
| 요소 | 권장 |
|------|------|
| 길이 | 12자 이상 |
| 대문자 | 최소 1개 |
| 소문자 | 최소 1개 |
| 숫자 | 최소 1개 |
| 특수문자 | 최소 1개 |

### 피해야 할 것
- 개인정보 (생일, 전화번호, 이름)
- 연속 숫자/문자 (1234, abcd)
- 키보드 패턴 (qwerty, asdf)
- 사전에 있는 단어
- 이전에 사용한 비밀번호

## 비밀번호 강도 측정

### 해킹 소요 시간 (대략)
| 비밀번호 유형 | 예상 시간 |
|---------------|-----------|
| 6자 숫자만 | 즉시 |
| 8자 소문자 | 2시간 |
| 8자 대소문자 | 1주 |
| 12자 대소문자+숫자 | 300년 |
| 16자 모든 문자 | 우주 나이 이상 |

## 비밀번호 생성 방법

### 1. 무작위 생성
가장 안전하지만 기억하기 어려움
```
예: Kj#9mP@2xLw!qR5n
```

### 2. 패스프레이즈 (Passphrase)
여러 단어를 조합하여 기억하기 쉽게
```
예: 보라색-코끼리-춤추다-7!
```

### 3. 문장 기반
문장의 앞글자 + 숫자 + 특수문자
```
"나는 2025년에 행복하게 살고 싶다"
→ N2025Hss!
```

## 비밀번호 관리

### 비밀번호 관리자 사용
- **1Password**: 유료, 기능 풍부
- **Bitwarden**: 무료/유료, 오픈소스
- **LastPass**: 무료/유료
- **삼성 패스**: 삼성 기기 내장

### 관리자 장점
1. 각 사이트별 다른 비밀번호
2. 자동 생성 및 자동 입력
3. 암호화된 저장
4. 여러 기기 동기화

## 2단계 인증 (2FA)

### 필수 설정 권장 계정
- 이메일 (가장 중요!)
- 은행/금융
- SNS
- 클라우드 스토리지
- 쇼핑몰

### 2FA 유형
| 유형 | 보안 수준 |
|------|-----------|
| SMS 인증 | 중 |
| 이메일 인증 | 중 |
| 앱 (OTP) | 상 |
| 하드웨어 키 | 최상 |

## 비밀번호 유출 확인

### 확인 방법
1. [Have I Been Pwned](https://haveibeenpwned.com) 접속
2. 이메일 주소 입력
3. 유출 여부 확인
4. 유출 시 즉시 비밀번호 변경
